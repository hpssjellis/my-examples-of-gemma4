📖 What is a token?

A token is roughly 3–4 characters or ¾ of a word in English. "Hello world" ≈ 2–3 tokens. A full paragraph ≈ 100–150 tokens.

Input tokens = your message + all previous conversation history sent to the model each time.
Output tokens = the new words Gemma generates in reply. "Max output tokens" caps this.

Gemma 4 E2B has a 128,000 token context window — the combined total of input + output it can handle at once. As your conversation grows, more of that window is used by history.

📷 Camera & Microphone (future feature)

Gemma 4 is multimodal — the E2B and E4B variants support image and audio inputs. A camera capture button is planned: it will grab a frame from your webcam via the Canvas API, convert it to a base64 data URI, and include it as part of the message to Gemma.

Microphone support requires a separate speech-to-text pipeline (e.g. Whisper via Transformers.js) to transcribe speech to text, which is then passed to Gemma. That pipeline can also run 100% locally.

TensorFlow.js note: Gemma 4 in Transformers.js uses the ONNX runtime — TF.js uses a different runtime and cannot run Gemma directly. However, TF.js could run alongside for preprocessing tasks (e.g. face detection, object classification) before passing results to Gemma.
