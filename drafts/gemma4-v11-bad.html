Gemma 4 · Transformers.js v4.0.1 — Teaching Demo

This page runs a large language model entirely in your browser — no server, no internet needed after the first download. The model is stored in your browser cache and loads instantly on future visits, even offline. Uses Gemma 4 E2B-IT (ONNX) via Transformers.js v4.

Important technical note for teachers: For multimodal (image) support, this page uses the low-level AutoProcessor + Gemma4ForConditionalGeneration API directly — the same approach shown in the official ONNX model card. The simpler pipeline() function cannot pass image data to this model correctly.

Step 1 — Choose settings, then click Load Model.

📖 What is a token? What is a context window?

A token is roughly 3–4 characters or about ¾ of an English word.
"Hello world" ≈ 2–3 tokens. A paragraph ≈ 100–150 tokens.

Input tokens = your message + all previous conversation history sent to the model each time.
Output tokens = the new words Gemma generates in reply. "Max output tokens" caps this.

Context window = total tokens (input + output) the model can hold at once. Gemma 4 E2B = 128,000 tokens. As your conversation grows, old messages eventually fall off.

tok/s = tokens per second. WebGPU + q4 → expect 10–40 tok/s depending on GPU. WASM/CPU → expect 1–5 tok/s. The live timer shows you the page is working, not frozen.

📷 Camera — capture an image to send with your message

Gemma 4 E2B is multimodal — it can see images alongside your text.

How it works technically: The browser draws a video frame onto a hidden Canvas, converts it to a base64 JPEG data URI, then passes it through AutoProcessor which converts it to pixel tensors the vision encoder understands. This uses Gemma4ForConditionalGeneration directly, not the simpler pipeline() API, because only the low-level API correctly handles image inputs for this model.

No image data ever leaves your browser — it goes directly to the local ONNX model.



🎤 Microphone — speak your message (Chrome on-device speech recognition)

Uses the browser's built-in Web Speech API (webkitSpeechRecognition) to transcribe your voice to text, which is then placed in the input box for Gemma.

In Chrome 139+, recognition uses SODA (Speech On-Device API) — a local model, no audio leaves your machine. Older Chrome sends audio to Google's servers. Firefox and Safari do not support this API.

Whisper (fully offline transcription via Transformers.js, ~150 MB) is a planned future addition. Gemma itself does not process raw audio in the ONNX version — it only sees the transcribed text.

🌐 HTML Viewer — when Gemma writes a webpage, render it here

Ask Gemma to write a single-file HTML page, then use these buttons to extract and preview it.

"View HTML" uses the regex /<!DOCTYPE html>[\s\S]*?<\/html>/si to strip Gemma's commentary and keep only the HTML document. Edit in the box below before rendering. Same pattern as the Chrome Built-in AI teaching page.
